# Ovk.Net

Попытка пересобрать OpenVK на ASP.NET Core, не выбрасывая по дороге старую базу, Chandler и, разумеется, фронтэнд (я думаю очевидно что проект этот написан по приколу, но кто знает, может когда нибудь и допилю до релиза)

Сейчас проект умеет жить на существующей MySQL-базе OpenVK и не требует ритуала с экспортом-импортом на каждый чих.

## Что уже переехало

- авторизация и регистрация, совместимые с хэшами Chandler;
- серверные сессии в `ChandlerTokens`;
- профили, редактирование страницы, статусы и короткие адреса;
- загрузка, показ и удаление аватаров через совместимую с OpenVK структуру альбомов и фотографий;
- друзья, заявки и подписки;
- стена и записи;
- личные сообщения, список диалогов и поиск сообщений;
- группы в базовом виде;
- настройки страницы, пароля, сессий и отображения аватаров;
- поиск людей и групп;
- «О сайте», правила, конфиденциальность, помощь, версия и прочее;

## Структура

```text
Ovk.Net/
  Ovk.Net.Core/           доменные модели и Chandler-совместимая безопасность
  Ovk.Net.Infrastructure/ EF Core, MySQL и additive-миграции legacy-схемы
  Ovk.Net.Web/            ASP.NET MVC, Razor, статика и контроллеры
```

## Что нужно для запуска

- .NET SDK 10;
- MySQL 8+ или совместимая MariaDB;
- база `openvk` — можно существующую от OpenVK;
- `make` по желанию. Все команды ниже имеют обычные `dotnet`-аналоги.

Строка подключения берётся из `Ovk.Net/Ovk.Net.Web/appsettings.json`. Для локальной разработки удобнее переопределить её переменной окружения, чтобы пароль не жил в коммите:

```powershell
$env:ConnectionStrings__DefaultConnection = "Server=localhost;Database=openvk;User=root;Password=ваш_пароль;SslMode=None;"
```

## Быстрый старт

```powershell
dotnet restore Ovk.Net/Ovk.Net.slnx
dotnet build Ovk.Net/Ovk.Net.slnx
dotnet run --project Ovk.Net/Ovk.Net.Web --launch-profile http
```

После запуска сайт обычно будет на `http://localhost:5038`.

Если установлен `make`, можно короче:

```powershell
make restore
make run
```

Полезные команды:

```powershell
make build          # собрать всё
make watch          # запустить с hot reload
make run-https      # HTTPS-профиль
make certs          # доверить локальный сертификат .NET
```

## Миграции legacy-базы

Здесь не используется обычная первая EF-миграция, которая попыталась бы создать заново уже существующие `profiles`, `posts` и `ChandlerUsers`. Вместо этого есть additive compatibility-migrator: он создаёт только отсутствующие таблицы/колонки и хранит применённые версии в `OvkNetSchemaMigrations`.

Прогнать только миграции без веб-сервера:

```powershell
dotnet run --project Ovk.Net/Ovk.Net.Web -c Release -- --migrate-only
```

Мигратор нужен, например, для таблиц медиа (`albums`, `album_relations`, `photos`) и части Chandler ACL. Он не делает `DROP TABLE`, потому что мы тут мигрируем соцсеть, а не вызываем демона форматирования диска.

## Аватары и файлы

Новые аватары пишутся в `php/storage/` в формате, совместимом с OpenVK. Это сделано специально: PHP-версия и ASP.NET-порт могут читать одну и ту же структуру файлов во время переезда.

## Лицензия

Исходный OpenVK распространяется по LGPL-2.1. Старые исходники остаются в каталоге `php/` локально; этот проект использует их как референс при поэтапном переносе на ASP.NET Core.
