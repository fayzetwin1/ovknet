# Ovk.Net

Попытка пересобрать OpenVK на ASP.NET Core, не выбрасывая по дороге старую базу, Chandler и узнаваемый интерфейс из нулевых. То есть это не «новая соцсеть с нуля», а аккуратная миграция: старый PHP-код лежит рядом как донор, а новое приложение постепенно учится делать то же самое на C#.

Сейчас проект умеет жить на существующей MySQL-базе OpenVK и не требует ритуала с экспортом-импортом на каждый чих.

## Что уже переехало

- авторизация и регистрация, совместимые с хэшами Chandler;
- серверные сессии в `ChandlerTokens`;
- профили, редактирование страницы, статусы и короткие адреса;
- загрузка, показ и удаление аватаров через совместимую с OpenVK структуру альбомов и фотографий;
- друзья, заявки и подписки;
- стена и записи;
- личные сообщения, список диалогов и поиск сообщений;
- группы в базовом виде;
- настройки страницы, пароля, сессий и отображения аватаров;
- поиск людей и групп;
- «О сайте», правила, конфиденциальность, помощь, версия и `robots.txt`;
- оригинальные CSS и изображения OpenVK, чтобы сайт не выглядел как очередной шаблон Bootstrap, который нашёлся в ящике стола.

## Что пока в процессе

Проект ещё мигрирует. В старом OpenVK гораздо больше функций: полноценные публичные страницы групп, фотографии и альбомы, вложения, комментарии, уведомления, приватность, видео, аудио, заметки, поддержка и админка. Они не забыты — просто не стоит тащить их одним огромным коммитом и потом искать, где всё сломалось.

## Структура

```text
Ovk.Net/
  Ovk.Net.Core/           доменные модели и Chandler-совместимая безопасность
  Ovk.Net.Infrastructure/ EF Core, MySQL и additive-миграции legacy-схемы
  Ovk.Net.Web/            ASP.NET MVC, Razor, статика и контроллеры

php/                      старый OpenVK на PHP, только источник для переноса
chandler/                 старый Chandler, тоже только источник для переноса
```

`php/` и `chandler/` специально добавлены в корневой `.gitignore`: они нужны локально, чтобы сверяться с оригинальным поведением и фронтендом, но в новый репозиторий не попадают.

Важно: если эти каталоги уже когда-то были добавлены в Git, одного `.gitignore` недостаточно. Один раз уберите их из индекса, не удаляя с диска:

```powershell
git rm -r --cached php chandler
```

## Что нужно для запуска

- .NET SDK 10;
- MySQL 8+ или совместимая MariaDB;
- база `openvk` — можно существующую от OpenVK;
- `make` по желанию. Все команды ниже имеют обычные `dotnet`-аналоги.

Строка подключения берётся из `Ovk.Net/Ovk.Net.Web/appsettings.json`. Для локальной разработки удобнее переопределить её переменной окружения, чтобы пароль не жил в коммите:

```powershell
$env:ConnectionStrings__DefaultConnection = "Server=localhost;Database=openvk;User=root;Password=ваш_пароль;SslMode=None;"
```

## Быстрый старт

```powershell
dotnet restore Ovk.Net/Ovk.Net.slnx
dotnet build Ovk.Net/Ovk.Net.slnx
dotnet run --project Ovk.Net/Ovk.Net.Web --launch-profile http
```

После запуска сайт обычно будет на `http://localhost:5038`.

Если установлен `make`, можно короче:

```powershell
make restore
make run
```

Полезные команды:

```powershell
make build          # собрать всё
make watch          # запустить с hot reload
make run-https      # HTTPS-профиль
make certs          # доверить локальный сертификат .NET
```

## Миграции legacy-базы

Здесь не используется обычная первая EF-миграция, которая попыталась бы создать заново уже существующие `profiles`, `posts` и `ChandlerUsers`. Вместо этого есть additive compatibility-migrator: он создаёт только отсутствующие таблицы/колонки и хранит применённые версии в `OvkNetSchemaMigrations`.

Прогнать только миграции без веб-сервера:

```powershell
dotnet run --project Ovk.Net/Ovk.Net.Web -c Release -- --migrate-only
```

Мигратор нужен, например, для таблиц медиа (`albums`, `album_relations`, `photos`) и части Chandler ACL. Он не делает `DROP TABLE`, потому что мы тут мигрируем соцсеть, а не вызываем демона форматирования диска.

## Аватары и файлы

Новые аватары пишутся в `php/storage/` в формате, совместимом с историческим OpenVK. Это сделано специально: PHP-версия и ASP.NET-порт могут читать одну и ту же структуру файлов во время переезда.

## Как переносить следующие куски

1. Найти контроллер/Presenter и Latte-шаблон в `php/`.
2. Посмотреть таблицы и модели, которые он использует.
3. Добавить C#-модель и additive-миграцию, если таблицы ещё нет.
4. Перенести поведение в контроллер и фронтенд в Razor, по возможности используя уже скопированные OpenVK CSS/ассеты.
5. Собрать Release и проверить маршрут.

Так меньше шансов однажды проснуться с 40 тысячами строк «временного» кода и одной кнопкой, которая работает только по вторникам.

## Лицензия и происхождение

Исходный OpenVK распространяется по LGPL-2.1. Старые исходники остаются в каталоге `php/`; этот проект использует их как референс при поэтапном переносе на ASP.NET Core.
